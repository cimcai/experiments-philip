Methodology

Ants are filmed from above as they push a red T-shaped load through a barrier. This viewer renders 4,635 ants detected and tracked across 211 s of footage. The pipeline below is built entirely from classical computer vision plus a graph-based identity solver — no learned model is used at any stage. Two heavier alternatives were evaluated and underperformed on this footage; see the final section.

1Detection — classical computer vision

Per-frame OpenCV pipeline operating on the cropped top half of the source frame. No neural network, no learned weights, no GPU.

  • Median background. 80 frames sampled uniformly across the full video; per-pixel grayscale median. Static elements (walls, printed dots, watermark) survive the median; moving ants and the slowly-moving T do not. Cached at pipeline/background.png.
  • Foreground mask. |gray − bg| > 12 → moving-pixel mask.
  • Adaptive threshold. cv2.adaptiveThreshold(MEAN_C, BINARY_INV, block=31, C=8) picks dark-on-light blobs.
  • Suppressions. Red-HSV ranges zero out the T-shape pixels; a fixed bounding box zeros out the corner watermark; AND-ed with the foreground mask.
  • Morphology. 2×2 opening drops single-pixel noise.
  • Connected components. Area filter 12–400 px². Below = noise; above = multi-ant clusters (handled in step 4).
  • Pose. Per blob, centroid + θ from second moments: θ = ½·atan2(2·cxy, cxx − cyy). Cheap orientation, no PCA call.

2Linker — Hungarian + constant velocity

Per frame: predict each track's next position from (x + vx·decay, y + vy·decay), build a cost matrix of Euclidean distances, cap each row by per-track radius, solve with scipy.optimize.linear_sum_assignment, update velocities with exponential blend. Unmatched detections start new tracks. Tracks unmatched for more than MAX_LOST frames are reaped.

3Stitcher — offline gap-bridging

The online linker rebirths a track once it's been lost for MAX_LOST frames. The offline stitcher reads the parquet and matches each track's tail (last-velocity × gap) to nearby track heads within STITCH_GAP_MAX = 60 frames and STITCH_DIST_MAX = 60 px, greedily merging via Union-Find. Reduces ID churn from honest occlusion gaps.

Source: ants-philip/pipeline/stitch_tracks.py

4Identity solver — tracklet graph with count conservation

The problem. Ants pile onto the T-shape and their bodies fuse into a single visual blob. The frame-to-frame linker can only see one detection where N ants exist, so it kills N − 1 tracks; when ants peel off again it has no memory of who entered, so each emergent ant gets a fresh ID. Before this stage the raw output contains 7,429 tracklet IDs for ~412 real ants — 18× fragmentation. 88.8% of births and 91.2% of deaths happen mid-frame at these pile-up events, not at video boundaries.

4a. Cluster-event detection

Any blob with area > 1.5× median ant area (~150 px²) is treated as a multi-ant cluster. Frames where K tracks die simultaneously next to a growing blob are recorded as a merge event; frames where K tracks are born next to a shrinking blob are a split event. Output: cluster_events.parquet.

4b. Tracklet graph

  • Nodes: each post-stitch tracklet, plus every cluster blob (the cluster is itself a node, carrying a member-count that changes over time).
  • Continuation edges: a parent track ends within 30 frames of a child track start and the gap is bridgeable by velocity extrapolation.
  • Merge-child edges: a track ends within 5 frames of a merge event, within the blob's radius.
  • Split-parent edges: a track starts within 5 frames of a split event, within the blob's radius.

4c. Greedy solver

  1. Every long-lived tracklet (> 10 s) is seeded with a unique global identity.
  2. Continuation edges merge same-ant fragments across honest motion gaps.
  3. Cluster events are walked in temporal order:
    • Merge: the cluster's "member set" is stamped with the identities entering it.
    • Split: each emergent track is assigned to the member identity whose predicted exit position and time-in-cluster best match.
  4. Conserve count: if K′ tracks emerge but only K < K′ entered, only K identities are assigned — the surplus stays as fragments. The solver does not fabricate identities to cover bookkeeping errors.

5Approaches that performed worse on this footage

Two heavier alternatives were evaluated head-to-head against the pipeline above on the same footage. Both were noticeably noisier in side-by-side comparison.

  • Pretrained video object-segmentation model with mask propagation. Seeded by drawing a bounding box around each ant on a single frame; the model then propagates a per-object mask forward through subsequent frames on a GPU. Strengths in principle: pixel-accurate masks and identity continuity through brief partial occlusion. Failures here: mask quality degraded on this footage's small, dark, near-identical ant bodies; identity still collapsed inside ant piles because the model has no count-conservation primitive; ants entering after the seed frame are invisible to it. Throughput: roughly minutes of GPU time per second of footage on a high-end accelerator.
  • Off-the-shelf multi-animal tracker with bundled background model. Tested on a wide range of organisms in the literature; emits one trajectory file per individual. On this clip the cached background was a poor fit for the lighting and scale, producing thousands of short fragmented tracks and false-positive trails on static scene features.

The takeaway: detector quality on this specific footage was the bottleneck, not feature richness. A specialist classical detector plus a cluster-aware identity solver produced cleaner per-frame output than either heavier alternative at a fraction of the compute cost.

6Reading this viewer

  • Each colored dot is one ant detection at the current frame. The color is hashed from the ant's resolved identity, so an ant keeps the same color across the whole video.
  • The number shown next to each ant in the sidebar is its resolved global identity from the solver above.
  • Trails draw the last N frames of motion behind each ant; selecting an ant draws its full path across the entire clip.
  • The translucent rectangle outlines the load (the T-shaped object). Force arrows draw briefly along an ant's heading whenever that ant is in contact with the load.
